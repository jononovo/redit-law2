Purpose & Capability
The name/description (agent wallet/credit card management) match the declared requirement for a single CREDITCLAW_API_KEY. Requiring only that API key is reasonable for the stated purpose. However, the SKILL.md instructs writing a config to ~/.openclaw/openclaw.json (setting snapshot defaults) even though the registry lists no required config paths — an omission/inconsistency to be aware of.

Instruction Scope
Runtime instructions direct the agent to retrieve one-time decryption keys, decrypt card details in-memory, and use browser automation to fill merchant checkout forms (incl. snapshots of arbitrary pages). These actions are necessary for the advertised functionality but involve handling highly sensitive payment data and interacting with arbitrary external sites. The skill also prescribes periodic polling (heartbeat) and instructions for saving webhook callback URLs and webhook secrets — actions that broaden the agent's scope and persistence.

What to consider before installing
What to consider before installing: - The skill legitimately needs and uses a CREDITCLAW_API_KEY — only provide it if you trust creditclaw.com. - The runtime instructions will cause the agent to retrieve one-time decryption keys and handle raw card data in memory, then fill external merchant checkout pages. This is necessary for the advertised capability but is intrinsically sensitive. Ensure you are comfortable with an agent having that runtime ability and that the owner’s approval mode is strict (ask_for_everything) until you trust it. - The SKILL.md tells you to save settings to ~/.openclaw/openclaw.json and to persist webhook secrets returned by the API — these file-write and secret-storage actions are not listed in the manifest. If you don’t want the skill to modify your OpenClaw config or store webhook secrets, do not follow those steps or inspect them carefully before saving. - Be cautious about callback_url/webhook settings: do not point them to untrusted servers (they can receive event payloads and may leak data). Treat any webhook_secret as a secret and store it securely if you accept the behaviour. - Recommended mitigations: test with a low-privilege/limited wallet (small balances and strict approval thresholds), keep owner approval required, review the service’s privacy/security policies, and rotate the API key if you suspect misuse. If you need more assurance, ask the publisher for an explicit declaration of any config paths the skill will write and for a security whitepaper or audited docs.
