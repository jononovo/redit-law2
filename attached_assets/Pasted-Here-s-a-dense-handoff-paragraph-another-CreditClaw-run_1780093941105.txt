Here's a dense handoff paragraph another CreditClaw-running agent could ingest:
CreditClaw is a financial-enablement layer that grants an autonomous agent owner-governed spending power via a REST API at https://creditclaw.com/api/v1, authenticated with a bearer CREDITCLAW_API_KEY that is bcrypt-hashed server-side and must never be transmitted to any non-creditclaw.com domain. Lifecycle: the bot self-registers (POST /bots/register) to receive an api_key plus a single-use claim_token, which the human redeems at /claim to bind the wallet to their account, add a funding method, and activate it; the agent then either consumes webhook events (HMAC-SHA256–signed via webhook_secret, verified on the X-CreditClaw-Signature header) or, webhook-less, long-polls GET /bot/messages and acks via POST /bot/messages/ack. It exposes multiple independent payment rails — primarily "My Card" (sub_agent_cards, owner card details delivered AES-256-GCM-encrypted via rail5.card.delivered, decrypted only in-memory per transaction) and a Stripe x402 USDC-on-Base stablecoin wallet — with GET /bot/status and GET /bot/wallet/spending as the canonical state/permission reads (cache ~30 min). Every purchase is gated server-side against the owner's approval_mode (default ask_for_everything), per-transaction/daily/monthly limits, human_approval_above_usd thresholds, and category allow/block lists, all non-bypassable client-side. The checkout flow itself (POST /bot/rail5/checkout → poll /checkout/status → one-time POST /bot/rail5/key → POST /bot/rail5/confirm) is documented in CHECKOUT-GUIDE.md, and in the v4 design card plaintext is isolated entirely from the agent's context: decryption and form-fill happen inside the Secure Fill Chrome extension, with the agent holding only a checkout_id and capturing the pay-button selector for a blind submit. New cards must pass a sandbox test purchase (pending_delivery → confirmed → active) before live use, and the agent should respect rate limits (e.g. status 6/hr, checkout 30/hr), honor the notes field as direct owner instructions, and surface low-balance/declined/frozen states back to the human.
