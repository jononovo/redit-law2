Your agent can now purchase from any Shopify store, worldwide, and pay with most methods the store accepts — such as card, Shop Pay, local payment methods, or bank transfer.

It also works across many other merchants, any site with a standard guest checkout, no login required. Coverage keeps expanding, so we'd recommend pointing it at the specific stores you care about to see how it performs, and for any that need a little steering you can guide the agent through their checkout. There are a ton of features that will ship over the next few weeks and months to improve the experience, and we are finally ready to have people like you break open agentic commerce.

How it works:

• Create a checkout with one request. Point it at the product and say what you want in plain language, e.g."buy size M, pay by card, use the cheapest delivery."
• Poll the checkout for its status while the agent works through the store's checkout pages.
• If you chose to pay by card, when it reaches payment it asks you for the card details (via a user action).
• You hand it a one-time-use Agent Card number and submit the payment. You get back a receipt.

Shipping over the next week:

• More descriptive progress events: some action events are still generic (you may see"Advancing checkout"); more granular, human-readable events land next week.
• Submit payment up front: the card is collected mid-run via a user action; soon you'll be able to pass the card in the initial request when you already know the number. (If a method needs an OTP, you'll still confirm it when we surface the awaiting-user-action event.)
• Sticky sessions: we don't support fully login-gated merchants yet, but sites that only need an OTP to proceed (e.g. OpenTable to book a table) already work (the agent logs in with the OTP during the run.) Right now the session isn't persisted, so you'll be asked for the OTP each time, but sticky sessions next week will fix that.
